## 1. 棋盘

### 1.1 棋盘结构

棋盘由 **60 个六边形格子**排列而成，形状为六角形（hexagonal grid）。每个格子使用**立方体坐标** (q, r, s) 标记，满足约束：

```
q + r + s = 0
```

坐标系示意（省略 s 轴，s = −q − r）：

```
         q →  -4  -3  -2  -1   0  +1  +2  +3
  r = -4:          ░░  ░░  ░░  ░░  ░░
  r = -3:       ░░  ░░  ░░  ░░  ░░  ░░
  r = -2:    ░░  ░░  ░░  ░░  ░░  ░░  ░░
  r = -1:       ░░  ░░  ░░  ░░  ░░  ░░
  r =  0:    ░░  ░░  ░░  ░░  ░░  ░░  ░░
  r = +1:       ░░  ░░  ░░  ░░  ░░  ░░
  r = +2:    ░░  ░░  ░░  ░░  ░░  ░░  ░░
```

（░░ = 六边形格子）

坐标范围由 `qAdjustments` 表确定（见 `core.py:141-144`）：

| q 值 | 调整后 r 范围 | 说明 |
|------|-------------|------|
| −4   | −2 ~ +3     | 偶数行 |
| −3   | −3 ~ +3     | 奇数行 |
| −2   | −2 ~ +3     | 偶数行 |
| −1   | −1 ~ +3     | 奇数行 |
|  0   | −2 ~ +3     | 偶数行 |
| +1   | −2 ~ +3     | 奇数行 |
| +2   | −1 ~ +3     | 偶数行 |
| +3   | −1 ~ +3     | 奇数行 |

> **注意**：JS 版本的 `qAdjustments` 使用字符串键，导致 `qAdjustments[2]`（数字键）查不到而返回 `undefined → 0`。Python 版本为与之对齐，使用字符串键 `"0"`, `"1"`, `"2"` 等，精确复现此行为。

### 1.2 格子分值

每个格子被随机分配一个分值：**1**、**2** 或 **3**。全局约束：

```
所有格子分值之和 = 100（固定 10 个 3 分、20 个 2 分、30 个 1 分）
3 分格子数量 = 8 ~ 10 个（随机）
其余格子 = 1 或 2 分（随机）
```

格子分值的三种状态：

| 状态 | `hex.value` | 说明 |
|------|-------------|------|
| 活跃   | `> 0`（1, 2, 3）| 可放置棋子或被计分 |
| 被占据 | `= 0`          | 有棋子在此格，不计分 |
| 已消除 | `= −1`         | 与所有棋子失去连接，已废弃 |

## 2. 棋子

### 2.1 棋子配置

游戏共有 **6 个棋子**，分属两位玩家：

| 棋子 ID | 归属 | 备注 |
|---------|------|------|
| 4, 6, 8 | Player 1（先手）| 偶数 ID |
| 5, 7, 9 | Player 2（后手）| 奇数 ID |

归属判断公式（`core.py:614`）：

```python
owner = piece.id % 2
# 偶数 → P1:  4%2=0, 6%2=0, 8%2=0
# 奇数 → P2:  5%2=1, 7%2=1, 9%2=1
```

棋子拥有以下属性（`core.py:64-69`）：

```python
@dataclass
class Piece:
    id: int              # 唯一标识
    hex: Optional[Hex]   # 当前位置（None = 未放置）
    hex_value: int       # 该格子原始分值（用于移动/销毁时还原）
    alive: bool          # 是否存活
```

## 3. 游戏阶段

游戏分为两个阶段，按顺序执行：

```
┌─────────────────────────────────────────────────────────────┐
│  放置阶段 (Placement)                                       │
│  6 步，双方严格交替各放 3 个棋子，Player 1 先手             │
│                                                             │
│  步骤 1: P1 放置（棋子 1）                                  │
│  步骤 2: P2 放置（棋子 1）                                  │
│  步骤 3: P1 放置（棋子 2）                                  │
│  步骤 4: P2 放置（棋子 2）                                  │
│  步骤 5: P1 放置（棋子 3）— P1 放满                         │
│  步骤 6: P2 放置（棋子 3）— P2 放满，所有棋子放置完毕       │
│  ↓                                                          │
│  步骤 7+: 进入移动阶段，Player 1 先手                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.1 放置阶段

**规则：**

1. Player 1 先手放置第一枚棋子
2. 双方**严格交替**轮流，各放置 3 枚
3. 只有**当前玩家自己的棋子**可以放置（其他玩家的棋子在 `get_legal_actions()` 中已被过滤）
4. 棋子放在**分值 < 3 的活跃格子**（`1 <= value <= 2` 且未被占据）。**3 分格子禁止在放置阶段占据**，但在移动阶段可以正常移入
5. 放置时，**该格子分值立即记入放置方得分**，格子变为"被占据"状态（`value = 0`）
6. 放置阶段在双方都放满 3 个棋子后结束（步骤 6 完成后），下一步自动进入移动阶段

**玩家切换逻辑（严格交替）：**

放置阶段中，每步结束后检查当前玩家是否已放满 3 个棋子：
- **未放满**：切换到对方（正常交替）
- **已放满**：当前玩家已无法继续放置，跳过切换
  - 对方若也未放满，则轮到对方继续放置
  - 双方都放满后 → 放置阶段结束，进入移动阶段

具体实现（`core.py:319-328`）：

```python
if self.phase == self.PHASE_PLACEMENT:
    current_placed = 已放置的当前玩家棋子数
    if current_placed < 3:
        切换到对方  # 正常交替
    # 已放满则不切换，等对方或进入移动阶段
```

这意味着交替顺序固定为：
```
P1 → P2 → P1 → P2 → P1 → P2 → (移动阶段 P1 先手)
```

### 3.2 移动阶段

**规则：**

1. 移动方选择一个己方棋子，向**同轴方向**移动任意步数
2. "同轴"的定义：目标格子与出发格子的 q轴 **或** r轴 **或** s轴 坐标相同
3. 路径上**不能有其他棋子**或**已消除格子**阻挡
4. 移动到达后，获得**目标格子**的分值（目标格子变为被占据 `value=0`）
5. 每次移动后，**与所有棋子失去连接的格子被消除**（`value = −1`）
6. 无合法移动目标的棋子被**移除**（棋子销毁，格子还原）

**棋子无移动时的处理（`core.py:436-439`）：**

```python
if not moves:
    self._destroy_piece(piece)   # 移除棋子，还原格子原值
    return -1.0                  # 惩罚 reward
```

## 4. 移动规则详解

### 4.1 六方向移动

六边形棋盘有 6 个邻接方向，使用立方体坐标表示（`core.py:32-39`）：

```
        (+1, -1, 0) ←→ (-1, +1, 0)     ←→ r 轴方向
      (+1, 0, -1) ←→ (-1, 0, +1)       ←→ s 轴方向
      (0, +1, -1) ←→ (0, -1, +1)       ←→ q 轴方向
```

### 4.2 移动步数计算

从 Hex A 到 Hex B 的距离（步数）定义为沿某轴需要移动的步数（`core.py:486`）：

```python
dq = to_q - from_q
dr = to_r - from_r
ds = to_s - from_s
steps = max(abs(dq), abs(dr), abs(ds))
```

> 注意：只有与当前棋子**同轴**的格子才是合法目标（同轴 = `dq=0` 或 `dr=0` 或 `ds=0`）。

### 4.3 路径检查

路径上每一个中间格子都必须满足（`core.py:480-503`）：

1. **存在**：在棋盘范围内（在 `_hex_map` 中）
2. **未被占据**：没有其他棋子
3. **活跃**：非已消除格子（`value > 0`）

### 4.4 合法移动示例

假设 Player 1 的棋子在 Hex(2, -1, -1)（q=2, r=-1, s=-1）上：

```
同轴候选格（q=2 或 r=-1 或 s=-1 的任意活跃空格）：

  q=2 的所有活跃空格 ←→ 沿 q 轴可移动
  r=-1 的所有活跃空格 ←→ 沿 r 轴可移动
  s=-1 的所有活跃空格 ←→ 沿 s 轴可移动

  （以上任意一条路径畅通即可）
```

## 5. 连通性消除

### 5.1 算法

每次移动阶段完成后（`core.py:516-546`），执行 flood fill：

```
对每个存活棋子的位置作为起点
  ↓
Flood fill 向 6 个方向扩展，只走 value > 0 的格子
  ↓
所有未能被任何 flood fill 到达的活跃格子
  → 设置为已消除（value = -1）
```

### 5.2 效果

连通性消除模拟"领地争夺"——远离棋子的边缘区域逐渐被消除，逼使双方棋子靠近，增加碰撞和策略对抗。

```
消除前:                      消除后（阴影=已消除）:

  棋子P1                         棋子P1
    ●                               ●
       ○ ○ ○ ← P2 棋子            ▓ ▓ ▓ ← P2 棋子
          ○ ○ ○                      ▓ ▓
    ↑                                    ↑
  两侧孤立的活跃格子会被消除        边缘格子已消除
```

## 6. 胜负判定

### 6.1 游戏结束触发条件

`core.py:557-578` 定义了两种结束情况：

| 条件 | 处理 |
|------|------|
| **一方棋子全灭** | 幸存方获得所有剩余活跃格子的分值 |
| **所有活跃格子被消除** | 双方都无棋可走，游戏结束 |

### 6.2 计分规则

最终得分 = 各项分值之和：

```
最终得分 = 放置阶段得分
        + 移动阶段所有得分
        + 游戏结束时剩余的活跃格子分值（若有）
```

> 重要：棋子被移除（无合法移动）时，**不额外扣分**，只损失未来得分机会。

### 6.3 胜负裁定

```
Player 1 得分 > Player 2 得分  →  Player 1 胜
Player 2 得分 > Player 1 得分  →  Player 2 胜
Player 1 得分 = Player 2 得分  →  平局
```

## 7. 回合流程总览

```
┌──────────────────────────────────────────────────────────┐
│  for each step:                                          │
│                                                          │
│  1. [放置/移动分发]                                       │
│     if 阶段 == 放置:                                      │
│         执行 _do_placement()                             │
│         （total_placed < 6 时执行，6 步后自动切换阶段）   │
│     else (移动阶段):                                      │
│         执行 _do_movement()                               │
│                                                          │
│  2. [步骤计数] episode_steps++                           │
│                                                          │
│  3. [连通性消除] (仅移动阶段)                             │
│     _eliminate_disconnected_hexes()                      │
│     - 所有无法从存活棋子 reach 的格子 → 已消除 (value=-1)│
│                                                          │
│  4. [游戏结束检查] _check_game_over()                     │
│     - 若有玩家全灭或无活跃格子 → 分配剩余分值 → 终止      │
│                                                          │
│  5. [玩家切换]                                            │
│     if 阶段 == 放置:                                      │
│         if 当前玩家已放满 3 个:                           │
│             不切换（等对方；若双方都放满则进入移动阶段）  │
│         else:                                             │
│             切换到对方                                    │
│     else:  (移动阶段)                                     │
│         切换到对方                                        │
│                                                          │
│  6. return observation, reward, terminated, info         │
└──────────────────────────────────────────────────────────┘
```

**放置阶段完整流程（6 步）：**

```
Step 1: P1 放置 → 切换检查: P1 只放了 1/3 → 切换到 P2
Step 2: P2 放置 → 切换检查: P2 只放了 1/3 → 切换到 P1
Step 3: P1 放置 → 切换检查: P1 只放了 2/3 → 切换到 P2
Step 4: P2 放置 → 切换检查: P2 只放了 2/3 → 切换到 P1
Step 5: P1 放置 → 切换检查: P1 已放满 3/3 → 不切换
Step 6: P2 放置 → 切换检查: P2 已放满 3/3，且双方都放满 → 放置阶段结束
Step 7: 进入移动阶段，current_player = 0 (P1 先手)
```

> **关键**：每步结束后**只检查当前玩家**是否放满，未放满则切换。不会因为一方放满就跳过另一方的放置机会。

## 8. 动作编码

在 Gymnasium 强化学习环境中，每个动作是一个 **整数 ID**（0 ~ 59）：

```
action_id = hexes 数组中的索引
```

| 阶段 | 合法动作含义 | 例子 |
|------|------------|------|
| 放置 | 可放置的空格子索引 | `action_id=12` → 放置到第 13 个格子 |
| 移动 | 己方棋子可到达的目标格子索引 | `action_id=31` → 移动到第 32 个格子 |

`get_legal_actions()` 返回所有当前合法动作 ID 的列表（`core.py:245-272`）。

## 9. Reward 设计

Reward 在 `penguinchess/reward.py` 中定义，支持两种模式：

### 稀疏 Reward（sparse）

```python
reward = 0                    # 游戏进行中
reward = +1.0                 # 我方获胜
reward = -1.0                 # 对方获胜
reward = 0                    # 平局
```

### 密集 Reward（dense）

```python
reward = score_gain / 100      # 每步得分归一化
reward -= 0.01               # 步数惩罚（鼓励快速结束）
```
